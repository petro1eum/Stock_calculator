# Техдолг

Короткий список того, что осталось проверить или доделать перед уверенным production-релизом.

## Перед выкатом

- Проверить 3-5 реальных SKU до/после унификации расчётов: `optQ`, `optValue`, `currentStock`, `safety`, `revenue12m`, ABC-категория.
- Прогнать реальный WB-импорт продаж, остатков и цен на свежей выгрузке.
- Сверить остатки WB с кабинетом: расчёт теперь берёт последние снапшоты по `sku + barcode + warehouse`, а не сумму истории.
- Обновить `Browserslist` / `caniuse-lite`, чтобы убрать предупреждение сборки.

## Модель спроса и сезонность

- Заменить текущий “ML” baseline на явную модель прогноза спроса или переименовать UI, чтобы не обещать ML.
- Добавить обучение сезонных коэффициентов из истории продаж вместо ручного ввода `monthlyFactors`.
- Зафиксировать контракт: ML-прогноз возвращает уже сезонный спрос или базовый спрос без сезонности, чтобы не было двойного умножения.
- Настроить реальное расписание обновления `wb_demand_forecast`; сейчас cron-миграция содержит заглушку `YOUR_USER_ID_HERE`.

## Бизнес-проверки

- Проверить валютную модель: UI в основном показывает рубли, но у товара всё ещё есть `currency` как метаданные закупки.
- Проверить портфельную оптимизацию на данных с нулевыми бюджетами, нулевой выручкой и очень малым числом SKU.
- Проверить поведение вкладки поставок: она всё ещё живёт в основном через локальные данные и не полностью связана с пересчётом товаров в реальном времени.

## Качество

- Добавить end-to-end сценарий: импорт WB -> пересчёт ассортимента -> Dashboard -> ABC -> сценарии.
- Добавить визуальную проверку основных вкладок после больших изменений расчётной логики.
